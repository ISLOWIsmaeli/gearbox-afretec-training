Module 2 Summary

Embedded Systems

Definition

An embedded system is a combination of computer hardware and software designed for a specific function.

Use Cases

  • Machines
  • Consumer electronics
  • Medical devices

Typical Structure of Embedded Systems

  • Microprocessor / Microcontroller
  • Input/Output Devices
  • Memory

Chip Manufacturers for Embedded Systems

  • Apple
  • IBM
  • Intel
  • Texas Instruments

Applications of Embedded Systems

Automobiles:

  • Cruise Control
  • Backup Sensor
  • Suspension Control
  • Navigation System
  • Alarm System
  • Airbag Systems

Mobile Phones:

  • Cameras
  • USB I/O Modules
  • OS

Other Applications:

  • Industrial Machines
  • Medical Equipment
  • Fitness Trackers

Typically, embedded systems are low-cost, low-power-consuming, small computers embedded in other mechanical or electrical systems.

Major Components of an Embedded System

  • Processor
  • Power Supply
  • Memory
  • Communication Ports

Firmware

Firmware is a type of software embedded into hardware for low-level control, acting as a bridge between hardware and software.

Characteristics of Embedded Systems

  • Consists of hardware, software, and firmware
  • Can be embedded in larger systems to perform specific tasks
  • Often use ASIC & FPGA SOC
  • Often used for sensing & real-time computing
  • Can vary in complexity & function
  • Often required to perform under a time constraint to keep the larger system performing properly
  • Can be microprocessor or microcontroller based

Structure of Embedded Systems

Hardware

  • Based around microprocessors & microcontrollers
  • Microcontrollers - All components built on one chip
  • Microprocessor - CPU integrated with other computing components

Software & Firmware

  • Varies in complexity
  • Requires little memory

Real-Time Operating Systems (RTOS)

Defines how the system works by supervising software & setting release schedules during program execution.

Hardware Components

  • Sensors - Convert physical sense to electrical signals
  • Analog-to-Digital Converters (ADC)
  • Processors - Process digital signals & store them in memory
  • Digital-to-Analog Converters (DAC)
  • Actuators - Convert electrical signals to physical signals

Types of Embedded Systems

Category 1

  1. Mobile Embedded Systems - Digital cameras, smartphones, laptops
  2. Networked Embedded Systems - Home security system, Point of Sale (POS) system
  3. Standalone Embedded Systems - Calculator, MP3 player
  4. Real-Time Embedded Systems - Traffic control systems, medical, military applications

Embedded System Architectures

  • Simple Control Loops - Subroutine managing specific parts
  • Interrupt-Controlled Systems - Main core with a secondary loop
  • Cooperative Multitasking

Debugging in Embedded Systems

  • Run software on PC emulating physical chip in software
  • Use General-Purpose Input/Output (GPIO) pin
  • Source-Level Debugger
  • Logic Analyzers

  • Edge Computing
